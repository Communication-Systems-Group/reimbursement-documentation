1.1 Registration

1.1.1 Step 1

The reimbursement-tool is connected to the UZH-IFI LDAP server. This allows employees of the IFI department to use their existing login-credentials provided by the University of Zurich.
On the initial login the user needs to pass a two-step registration process. See figure 1.1 for the first step. Here, the user needs to input two values:

  • UZH personnel number is the employee number, written on the employment card. It is of the form 01-123-456. Enter the number without the leading 0 and hyphens; i.e. 123456.
  • UZH phone number is the personnel phone number of the employee. Please enter it in the form 0441234567.

Furthermore the user can select his preferred language. However, the language will be automatically determined based on the browser language settings.

An account can also be deactivated, so that the user cannot participate actively in the reimbursement-process.


Figure 1.1: Registration step one: Capture personnel data

1.1.2 Step 2

On figure 1.2 the user needs to add his handwritten signature either by uploading an image or capture it using a third party touchscreen device, i.e. a smart phone. This signature is required later on to stamp it onto the generated Pdf document containing all receipts.


Figure 1.2: Registration step two: Capture signature

After completing the registration process, the captured information and signature can always be edited by navigation to the Settings on the navigation bar.

1.2 Expenses

An expense is an entity captured by a Registered user. Every expense is defined by an SAP description that will be visual in SAP. An expense consists of 1 to 15 receipts.

The Registered user has to create an expense, add receipts to it and submit it to a Manager. The system selects the respective Manager automatically given that is defined. The Manager will check the expense for correctness, add the corresponding projects and submit it to be approved by the Finance management. For a detailed description of the overall reimbursement-process refer to section 1.7.

1.2.1 Create Expense

To create an expense, the user needs to navigate to the Dashboard and click on the button Create new Expense. A modal-window will appear to enter the SAP description. Once it is entered correctly and the user clicked Next an empty expense will be created. The user can start with adding receipts (see figure 1.3).


Figure 1.3: Expense-View: Overview

On the top left-hand side the SAP description is added. The small Edit button on the right can be used for editing. On the top right-hand side the current process step and the unprocessed steps of the expense are visualized. This process bar always indicates the current state of the expense.
Further the user can add, edit and remove receipts. To add a new receipt the user needs to click on the button Add new Receipt. See section 1.2.2 for details.
Existing receipts can be edited and removed by clicking on the respective buttons.

1.2.2 Receipt

By adding a new receipt or editing an existing one, the user needs to fill out the following mandatory fields:

  • Receipt date will be used to calculate the correct exchange rate if a foreign currency is selected.
  • Cost category selected based on a pre-defined list of available cost categories.
  • Original and calculated amount based on the exchange rate. The rate used in the system is based on the market rate plus a 2% variation immunity.
  • The project assignment is added by a user with role Manager or Department manager.
  • A short explanation that describes the purpose of the receipt.
  • Receipt attachment consists of an uploaded document that verifies the input information like receipt date, price and currency.

See figure 1.4.

1.2.3 Assign Expense to Manager

If all receipts are captured, the expense will be submitted to a Manager. Once it is submitted, it cannot be edited by the Creator anymore.

1.2.4 Sign document

The signing is required for the generation of the Pdf that contains all of the captured receipts of the expense. This Pdf uses the signatures of the respective Creator, Manager and Finance management to proof their acceptance of the expense and the captured receipts. During the signing process the user will be prompted to select his preferred signing method (see figure 3.5). The two different types available will be explained in detail:

  • A Digital signature ensure the authenticity of the signer. Any changes made to the document after it has been signed will invalidate the signature. To use it the user has to provide his private key to successfully sign the document.
  • Electronic signature does not ensure the authenticity of the signer. Anyone can theoretically make changes on the document after it has been signed without the signature become invalid. The handwritten signature captured at the registration (see section 1.1) will be stamped on the Pdf document.


Figure 1.5: Expense: Sign document

Please note, that if a signing method has been selected all subsequent signers, like Manager and Finance management have to apply the same method.

1.3 User roles

Every user that exists in the LDAP of the University of Zurich is capable to login to the system. He can use the same user name and password that he uses for the other University of Zurich services for login.

The reimbursement-tool provides different user roles. The roles on the reimbursement-tool are mapped with the LDAP user roles of the University of Zurich. So a user who is defined as professor for a specific group on the LDAP will be the manager for all users of this group. The reimbursement-tool provides the following roles:

  • Unregistered user are users, who are authorized to login the reimbursement-tool but have not yet completed the registration process, see section 1.1.
  • Registered user / User passed the registration process. He can access the reimbursement-tool and create and manage his created expenses. He has one of the following roles:
    • Creator iis authorized to create, edit and print expenses as long as they are in DRAFT state. He’s the initial creator of an expense.
    • Manager / Professor are authorized to reject, accept and edit expenses as long as they are assigned to him.
    • Department manager has the same authorizations as the Manager. If a Manager is outreach, his assignments will be forwarded to the respective Department manager.
    • Finance management is authorized to reject, accept and edit expenses once they have been accepted by a Manager or Department manager. Furthermore they can manage the available cost categories and search for expenses.
    • Head of institute has the same authorizations as the Manager. In contrast to the Department manager there exists only one Head of institute.

1.4 Guest view

The guest view allows users to retrieve expenses without the need of a user with role Registered user for the reimbursement-tool. They can open the guest view using a special URL. It is provided on every printed expense document (see figure 1.6). The URL is located on the lower right side of the page in form of a QR-Code and an URL. So that the user can either scan the QR-Code or typewrite the URL in a browser window and he’ll get access to the respective expense.

1.6 Retrieve Expenses

1.6.1 Search for Expenses

For the Finance management users a sophisticated search interface for expenses exist; the Admin Pool Search (see figure 1.8). The search result will be limited by the following values:

  • Specify the date range the expense has been created.
  • Last name of the expense applicant.
  • The permission / role of the applicant.
  • The current state of the expense.
  • Based on the selected cost category of the receipt.
  • The accounting description (similar to the SAP description).


Figure 1.8: Administration: Search for Expenses

The Finance management users are authorized to execute specific actions to change the expense state:

  • Reassign: If the expense has been already printed by the user, it can be reassigned to a Finance management user.
  • Reset: has mistakenly submitted the expense, it can be reset and assigned back to him.

1.6.2 Archive

Besides the Admin Pool Search there exists an archive page for all users. The archive provides a list of all expenses in PRINTED state that the current, logged-in user has created.

1.6.3 Analytics

The reimbursement-tool provides an analytics-view for the Finance management. It should help to empower their knowledge about the overall amount of the captured expenses, rate of printed expenses as well as the actual state-distribution among the expenses.

1.7 Expense process

The entire process is basically divided into three iterations. The 1. iteration; expense creation and confirmation, 2. iteration; the expense signing, 3. iteration; print and carry the documents to the Finance management of the University of Zurich.
Expense date, cost category, amount, explanation and attachment will be captured by the user, who is the expense Creator. As soon as he’s done with capturing and submitting the expense, it’ll be assigned to the Manager. The Manager reviews the expense, adds the respective projects and assigns the expense to the next instance for review; the Finance management. If they are satisfied, they submit it and the expense moves into the second iteration; the signing. For more detailed information containing the signing refer to section 1.2.4.


Figure 1.9: Expense: Process of expense creation

After the signing is completed by all instances. The expense will be available to print by the Creator. As soon as all the documents are printed, they need to be carried to a Finance management user for further processing. The Finance management will send all relevant documents to the Finance Administration of the University of Zurich. This action completes the process.
